单例模式

为全局提供单一的实例

包含：私有构造函数，私有静态变量，公有静态方法
作用：
    私有构造函数：确保不被外界所创建
    私有静态变量：声明单一实例
    公有静态方法：公外界获取实例

创建的几种方式：
    懒汉式：
        1.线程不安全的[LazyNotSafeSingleton]---当多个线程同时访问且实例为空时，可能创建多个实例
        2.线程安全的[LazySafeSingleton]---增加synchronized独占锁，确保实例只初始化一次，但是当一个线程访问时，其他线程都需要等待，这样对性能有一定的损耗
    饿汉式：
        1.线程安全[HungrySafeSingleton]--虽然线程安全，但直接实例化的方式也丢失了延迟实例化带来的节约资源的优势--实例应该在需要的时候被初始化
    双重校验锁：
        1.线程安全[DoubleCheckSafeSingleton]--实例中最符合的方式--加锁是在未实例化时，才加锁，初始化之后，多线程再获取，不需要加锁，及节省资源又提高性能
